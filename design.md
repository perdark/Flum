To create an effective PRD or brief, I need to understand:

*   **What problem are we trying to solve?**
*   **Who is the target user?**
*   **What's the core idea or solution?**
*   **What business goals does this project support?**
*   **Are there any existing systems or data involved?**
*   **What's the desired outcome?**

**However, I can provide you with a comprehensive TEMPLATE for a Project PRD/Brief.** You can then fill this out with your specific project details.

This template is designed to be flexible. For a "brief," you'd focus on the high-level sections (I-VII and X). For a full "PRD," you'd dive into the detail of every section.
